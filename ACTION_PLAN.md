# Atlan Performance — Initial Setup Action Plan

Derived from `atlan-initial-project-setup-prompt.md` (primary) and `docs/` (depth). Scope = the first buildable milestone only: architecture, navigation, design system, shared domain, mock offline data, 10 starter screens, starter tests, docs. **Not** the full product.

## Guiding constraints (carry through every phase)
- KMP shared core + native SwiftUI (iOS) + native Compose (Android). No cross-platform UI.
- Local data is source of truth; network never blocks Wet Mode. Repos are fake/in-memory but shaped for later real-DB swap.
- No streaks/leaderboards/peer rank/red missed-session/"behind plan" anywhere (UI, copy, telemetry, model fields). ES+EN first-class.
- Use exact names/packages (CLAUDE.md → Naming). Use verbatim starter copy (EN+ES) from the prompt.

## Offline persistence core (Phase 2 — started)
- [x] **SQLDelight foundation, verified on all three platforms.** Wired the `app.cash.sqldelight` plugin + drivers into the version catalog and shared `build.gradle.kts`; declared it at the root with `apply false` to stop its transitive Kotlin Gradle plugin from **downgrading** KGP (that was the one real snag — it broke the `compilerOptions { jvmTarget }` DSL until pinned). Schema in `SyncQueue.sq` → generated `AtlanDatabase`. Cross-platform `expect/actual DatabaseDriverFactory` (Android `AndroidSqliteDriver(context)`, iOS `NativeSqliteDriver`, JVM `JdbcSqliteDriver(IN_MEMORY)`); iOS links system `libsqlite3` (`-lsqlite3` in `project.yml` — the static framework leaves `sqlite3_*` undefined otherwise).
- [x] **Sync queue migrated to durable storage** (`SqlDelightSyncQueueRepository` replaces `FakeSyncQueueRepository` in `AtlanShared`; `AtlanShared(databaseDriverFactory)` now takes a driver, provided by each platform). `markSynced` is the drain primitive. Payload `Map<String,String>` encoded with RS/US separators (dependency-free).
- [x] **Verified:** `:shared:jvmTest` (6 tests — 4 original + new `SyncQueuePersistenceTest` proving an item is readable from a *second* repo on the same DB + drops out of `pending()` once synced); `:androidApp:compileDebugKotlin`; **iOS `xcodebuild` BUILD SUCCEEDED** (sim) with the native SQLite driver linked.

- [x] **Active-session persistence → Resume / Recovery (both platforms, verified green).** New `sessionProgress` SQLDelight table + `SessionProgressRepository` (SQLDelight impl) + use cases (`SaveSessionProgress`, `LoadSessionProgress`, `ClearSessionProgress`, and a pure `ResumeWorkoutTimerUseCase`). Wet Mode now **saves a snapshot at set granularity** (on set-complete/pause/rest) and **clears it on completion** (full or ended-early), so a snapshot lingers only when the app was left/killed mid-session. The Today Dashboard shows a calm **Resume / Discard** banner ("Resume session · Set X of N") when a snapshot exists; Resume rebuilds the timer at the interrupted set with prior sets still counted (the interrupted set starts fresh — no time accrues while closed, correct for intervals). Surfaced as a dashboard banner rather than a launch interstitial (promote-to-launch is a small follow-up). New `SessionProgressPersistenceTest` proves save→(fresh repo on same DB)→load→resume→clear. **Verified:** `:shared:jvmTest` (7 tests) + `:androidApp:compileDebugKotlin` + iOS `xcodebuild` (sim) all green.

- [x] **Workout History (SQLDelight-backed, both platforms, verified green).** New `completedSession` table + `WorkoutHistoryRepository` (SQLDelight impl) + `RecordCompletedSession` / `GetWorkoutHistory` use cases. Wet Mode records a finished session on **Done** (title, completed/total sets, total elapsed, optional perceived-effort key, ISO timestamp, full/partial flag). New **History** screen (Settings → "Workout history") lists sessions newest-first with calm "Partial"/effort pills + a reassuring empty state. New `WorkoutHistoryPersistenceTest` (record full + partial → fresh repo on same DB → newest-first, partial flagged, effort round-trips). **Verified:** `:shared:jvmTest` (8 tests) + `:androidApp:compileDebugKotlin` + iOS `xcodebuild` (sim) all green. **Scoping note:** persisted *user-generated* data (sync queue, session progress, history) now lives in SQLDelight; read-only **seed** repos (plan/why/profile defaults) stay in-memory fakes until a real backend hydrates them — rewriting them now would be churn without user value.

- [x] **Sync drain engine + background workers (both platforms, verified green).** `DrainSyncQueueUseCase` + a pluggable `SyncUploader` (the backend seam); the only impl this milestone is `SimulatedSyncUploader` (no backend stand-in). Android **`SyncWorker`** (`CoroutineWorker` + `androidx.work`) opens the same DB and drains, scheduled on launch with a `CONNECTED` constraint; iOS **drains on foreground** (`scenePhase == .active`) with a documented `BackgroundSync` (BGTaskScheduler) scaffold for the production background path (not registered yet — needs an Info.plist id). The Wet Mode summary's **Retry** now runs the real engine (→ calm "Synced" on a full drain). New `SyncDrainTest` (drains when accepted; items stay pending — never dropped — when not). **Verified:** `:shared:jvmTest` (10 tests) + `:androidApp:compileDebugKotlin` + iOS `xcodebuild` (sim) all green.
